### 1. وابستگی‌های گمشده
- **مشکل:** پکیج `email-validator` نصب نشده بود
- **راه‌حل:** نصب پکیج‌های `email-validator` و `dnspython`
- **وضعیت:** ✅ برطرف شد

### 2. جداول دیتابیس ایجاد نشده بودند
- **مشکل:** دیتابیس PostgreSQL هیچ جدولی نداشت (relation "users" does not exist)
- **راه‌حل:**
  - ایجاد migration اولیه `001_initial_setup.py` برای ایجاد تمام جداول پایه
  - به‌روزرسانی زنجیره migration به ترتیب صحیح
  - اجرای `alembic upgrade head` برای ایجاد تمام جداول
- **وضعیت:** ✅ برطرف شد

### 3. عدم وجود کاربر ادمین
- **مشکل:** هیچ کاربر ادمین برای تست وجود نداشت
- **راه‌حل:** اجرای `create_admin.py` برای ایجاد کاربر ادمین پیش‌فرض
  - نام کاربری: admin
  - رمز عبور: admin123
- **وضعیت:** ✅ برطرف شد

### 4. عدم تطابق فرمت پاسخ
- **مشکل:** endpointهای login و /me داده‌ها را در ساختار تو در تو برمی‌گرداندند
- **راه‌حل:** به‌روزرسانی فرمت پاسخ برای تطابق با انتظارات تست
  - تغییر پاسخ login از `{"data": {"token": ..., "user": ...}}` به `{"token": ..., "user": ...}`
  - تغییر پاسخ /me از `{"data": {...}}` به ساختار تخت
- **وضعیت:** ✅ برطرف شد

### 5. فرمت پاسخ لیست کاربران
- **مشکل:** endpoint دریافت کاربران `{"data": [...]}` برمی‌گرداند اما تست انتظار `{"users": [...]}` را داشت
- **راه‌حل:** افزودن `response['users'] = response['data']` برای شامل شدن هر دو کلید
- **وضعیت:** ✅ برطرف شد
